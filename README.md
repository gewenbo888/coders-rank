## Scoring methodology

Each tool/model scored **1–10 across 6 criteria**, weighted into a single **Overall Score (0–100)**:

| Criterion | Weight | Why |
|---|---:|---|
| **Code generation quality** | **25%** | The whole point of the tool. Highest weight. |
| **Context understanding** | **20%** | Huge differentiator in 2026 — large + smart context windows separate leaders. |
| Developer productivity impact | 18% | What actually shows up in your week. |
| Pricing | 13% | Important but not decisive at pro tier. |
| IDE integration | 12% | Where the work happens. |
| Speed and latency | 12% | Fast feedback loops matter. |

## Three sortable columns

- **Overall** — composite weighted score (0-100)
- **Code Quality** — code-generation quality score (×10)
- **Productivity** — productivity-impact score (×10)

Sort by **Code Quality** to see Claude Code, DeepSeek V4, Cursor at the top. Sort by **Productivity** for the "ship more this week" view. Sort by **Overall** for the balanced view.
